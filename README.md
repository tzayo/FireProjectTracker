# 🚒 מערכת ניהול כיבוי אש - קיבוץ גלאון

מערכת ניהול מקיפה למחלקת כיבוי האש בקיבוץ גלאון, כוללת מעקב אחר צוותים, הידרנטים, ציוד, משימות ותחזוקה.

## 🎯 תכונות

### ניהול צוותים
- מעקב אחר צוותי כיבוי אש זמינים
- ניהול חברי צוות ומפקדים
- מעקב סטטוס: זמין, בשירות, לא זמין
- פרטי קשר ומידע נוסף

### מיפוי הידרנטים
- מפה אינטראקטיבית של כל ההידרנטים בקיבוץ
- מעקב מיקומים עם קואורדינטות GPS
- סטטוס תקינות: תקין, דורש תחזוקה, לא פעיל
- מידע על לחץ מים ותאריכי בדיקה אחרונה

### ארונות ציוד חירום
- מיפוי מלא של ארונות הציוד
- רשימת ציוד מפורטת לכל ארון
- סטטוס מוכנות
- מעקב בדיקות תקופתיות

### ניהול משימות
- מעקב משימות רבעוניות
- הגדרת עדיפויות: נמוכה, בינונית, גבוהה, דחופה
- סוגי משימות: בדיקה, תחזוקה, אימון, רבעונית
- הקצאת משימות לצוותים
- מעקב התקדמות והשלמה

### רשומות תחזוקה
- תיעוד כל פעולות התחזוקה
- מעקב עלויות
- היסטוריה מלאה לכל פריט
- סינון לפי סוג פריט ותאריך

### לוח בקרה
- סקירה כללית של המערכת
- סטטיסטיקות בזמן אמת
- התראות על משימות דחופות
- מעקב אחר פריטים הדורשים תחזוקה

## 🛠️ טכנולוגיות

### Backend
- **Python 3.x** - שפת התכנות
- **Flask** - Web Framework
- **SQLAlchemy** - ORM למסד הנתונים
- **SQLite** - מסד נתונים מקומי
- **Flask-CORS** - תמיכה ב-CORS

### Frontend
- **React 18** - ספריית UI
- **React Router** - ניתוב
- **Axios** - HTTP Client
- **Leaflet & React-Leaflet** - מפות אינטראקטיביות
- **Tailwind CSS** - עיצוב
- **RTL Support** - תמיכה מלאה בעברית

## 📋 דרישות מקדימות

- Python 3.8 ומעלה
- Node.js 14 ומעלה
- npm או yarn

## 🚀 התקנה והפעלה

### שיטה 1: שימוש בסקריפט האוטומטי (מומלץ)

```bash
# הרץ את סקריפט ההתקנה
./start.sh
```

הסקריפט יבצע אוטומטית:
1. בדיקת התקנת Python ו-Node.js
2. יצירת סביבה וירטואלית לפייתון
3. התקנת כל התלויות
4. אתחול מסד הנתונים
5. הכנת המערכת להרצה

### שיטה 2: התקנה ידנית

#### Backend

```bash
# עבור לתיקיית Backend
cd backend

# צור סביבה וירטואלית
python3 -m venv venv

# הפעל את הסביבה הוירטואלית
# Linux/Mac:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# התקן תלויות
pip install -r requirements.txt

# הרץ את השרת
python3 app.py
```

השרת יעלה על: `http://localhost:5000`

#### Frontend

```bash
# פתח טרמינל חדש ועבור לתיקיית Frontend
cd frontend

# התקן תלויות
npm install

# הוסף את Tailwind CSS
npm install -D tailwindcss postcss autoprefixer

# הרץ את אפליקציית React
npm start
```

האפליקציה תיפתח אוטומטית ב: `http://localhost:3000`

## 📖 שימוש במערכת

### ניהול צוותים
1. נווט ללשונית "צוותים"
2. לחץ על "הוסף צוות חדש"
3. מלא את פרטי הצוות: שם, מפקד, חברים, סטטוס, טלפון
4. שמור את הצוות

### הוספת הידרנט
1. נווט ללשונית "הידרנטים"
2. לחץ על "הוסף הידרנט"
3. מלא פרטים: שם, מיקום, קואורדינטות, סטטוס, לחץ
4. הצג במפה או ברשימה

### יצירת משימה רבעונית
1. נווט ללשונית "משימות"
2. לחץ על "הוסף משימה"
3. בחר "רבעונית" כסוג משימה
4. בחר רבעון ושנה
5. הגדר עדיפות ותאריך יעד
6. הקצה לצוות

### תיעוד תחזוקה
1. נווט ללשונית "תחזוקה"
2. לחץ על "הוסף רשומה"
3. בחר סוג פריט (הידרנט, ארון ציוד, וכו')
4. תאר את העבודה שבוצעה
5. הוסף עלות ומבצע

## 🔌 API Endpoints

### Teams
- `GET /api/teams` - קבל את כל הצוותים
- `POST /api/teams` - צור צוות חדש
- `GET /api/teams/:id` - קבל צוות ספציפי
- `PUT /api/teams/:id` - עדכן צוות
- `DELETE /api/teams/:id` - מחק צוות

### Hydrants
- `GET /api/hydrants` - קבל את כל ההידרנטים
- `POST /api/hydrants` - צור הידרנט חדש
- `GET /api/hydrants/:id` - קבל הידרנט ספציפי
- `PUT /api/hydrants/:id` - עדכן הידרנט
- `DELETE /api/hydrants/:id` - מחק הידרנט

### Equipment Cabinets
- `GET /api/equipment-cabinets` - קבל את כל ארונות הציוד
- `POST /api/equipment-cabinets` - צור ארון ציוד חדש
- `GET /api/equipment-cabinets/:id` - קבל ארון ספציפי
- `PUT /api/equipment-cabinets/:id` - עדכן ארון ציוד
- `DELETE /api/equipment-cabinets/:id` - מחק ארון ציוד

### Tasks
- `GET /api/tasks?quarter=Q1&year=2025&status=pending` - קבל משימות עם סינון
- `POST /api/tasks` - צור משימה חדשה
- `GET /api/tasks/:id` - קבל משימה ספציפית
- `PUT /api/tasks/:id` - עדכן משימה
- `DELETE /api/tasks/:id` - מחק משימה

### Maintenance
- `GET /api/maintenance?item_type=hydrant` - קבל רשומות תחזוקה
- `POST /api/maintenance` - צור רשומת תחזוקה
- `GET /api/maintenance/:id` - קבל רשומה ספציפית
- `PUT /api/maintenance/:id` - עדכן רשומה
- `DELETE /api/maintenance/:id` - מחק רשומה

### Dashboard
- `GET /api/dashboard/stats` - קבל סטטיסטיקות למערכת
